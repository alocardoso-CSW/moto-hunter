§3 Filters

Every watch is defined by these fields, all optional except brand and model:

  • Brand — e.g. Yamaha
  • Model — e.g. MT-07
  • Price — min / max, in €
  • Year — min / max
  • Km — min / max
  • Location — optional; unset searches nationwide

Listings that fail a filter are still stored in the background — they remain useful market context for scoring the ones that pass, but won't clutter your view of active matches.

§4 Sources — Portugal

Each marketplace gets its own scraper behind one common interface: fetch(filters) → list[Listing]. A new source is an isolated addition, not a rewrite.

Primary
Standvirtual.pt

Largest PT marketplace. Listing data is often embedded as structured JSON — more reliable to parse than raw HTML.

Primary
OLX.pt

Large general classifieds site with strong motorcycle presence.

Secondary
CustoJusto.pt

Smaller reach, but occasionally surfaces listings the other two miss.

Secondary
Auto.pt

Adding as requested. To confirm during build: its motorcycle ("motos") catalog coverage looks thinner than cars — will verify listing volume before relying on it.

Optional · best-effort
Facebook Marketplace

Off by default, enabled per watch. Real caveat: FB has no public search API, requires a logged-in session, aggressively blocks automated access, and its ToS prohibits scraping. Feasible as a low-frequency, best-effort source using your own session — but expect it to break more often than the others, with a small risk of the account being flagged. Turn it on per watch only if the other four aren't enough.

§6 Data model

Two tables now instead of one flat list: what you're watching, and what's been found.

  • Watch — brand, model, price/year/km min–max, location, active sources, last_run_at
  • Listing — watch_id, source, external id, title, price, year, km, cc, power, fuel, location, url, image_url, first_seen, last_seen, is_active
  • PriceHistory — every price change per listing, timestamped, to catch reductions between runs

§7 Scoring — self-calibrating, no thresholds to tune

Every listing is ranked only against other active listings for that same watch — an MT-07 is never compared against a CB500F.

  • Price percentile within the watch's currently active pool, loosely adjusted for year/km outliers
  • Year and mileage as secondary factors
  • Price-drop detection — a reduced price is a strong signal independent of percentile, visible across runs whenever you trigger them
Great deal Good Fair Overpriced

Your filters decide what counts as a match; scoring decides how good each match is — the two operate independently, per watch. The badge shown on each dashboard card (§2) is this bucket.

§8 Hosting — decided: Oracle Cloud Always Free

You asked for fully free with no compromises, so this is now a decision rather than an open question.

  • Oracle Cloud "Always Free" VM — a real, small Linux VM that is genuinely free forever, no time limit, no credit-card-triggered surprise charges, and no spin-down. The app and its SQLite file just run on it continuously, exactly like a machine you own.
  • Cost: one-time manual setup — SSH in once, install Python, run the app as a background service (systemd) so it survives reboots. After that, zero ongoing maintenance and zero recurring cost.
  • Why not Render/Fly free tiers: both have drifted toward time-boxed free databases or credit-card-gated free tiers — not the "fully free" guarantee you asked for. Oracle's Always Free tier has stayed a stable, uncapped offering.
  • Repo visibility (public/private) is unaffected either way — no dependency on GitHub Pages or Actions.

§9 Notifications

Still out of scope for v1, by request — the dashboard is the interface for now.

notify.py stays stubbed into the architecture on purpose, so email (or Telegram, etc.) slots in later without restructuring anything upstream of it.

§10 Scraping etiquette

Low, predictable load: a handful of searches per watch, only when you trigger a run, with realistic delays and a normal user-agent. Personal-use monitoring of public listing pages, not bulk harvesting — and the one source that carries real ToS risk (Facebook) is opt-in and off by default.
